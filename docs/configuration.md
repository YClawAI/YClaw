# Configuration Reference

YCLAW is configured through `yclaw.config.yaml`, generated by `yclaw init`. This document covers every field.

---

## File Overview

| File | Purpose | Created By |
|------|---------|-----------|
| `yclaw.config.yaml` | Infrastructure, channels, deployment | `yclaw init` |
| `.env` | Credentials and secrets | `yclaw init` |
| `.yclaw-cli.json` | CLI metadata (deployment target, LLM) | `yclaw init` |

---

## Full Schema

```yaml
# -- Storage ---------------------------------------------------

storage:
  state:
    type: mongodb                 # Only option currently
    uri: mongodb://localhost:27017  # Optional -- falls back to MONGODB_URI env
    database: yclaw_agents        # Optional -- falls back to MONGODB_DB env

  events:
    type: redis                   # Only option currently
    url: redis://localhost:6379   # Optional -- falls back to REDIS_URL env

  memory:
    type: postgresql              # Only option currently
    url: postgresql://...         # Optional -- falls back to MEMORY_DATABASE_URL env

  objects:
    type: local                   # local | s3
    # If type: local
    path: ./data/objects          # Optional -- falls back to YCLAW_OBJECT_STORE_PATH env
    # If type: s3
    # bucket: my-bucket           # Optional -- falls back to YCLAW_S3_BUCKET env
    # prefix: yclaw/              # Optional
    # region: us-east-1           # Optional

# -- Secrets ---------------------------------------------------

secrets:
  provider: env                   # env | aws-secrets-manager
  # If provider: aws-secrets-manager
  # prefix: yclaw/                # Optional -- prefix for secret names
  # region: us-east-1             # Optional

# -- Channels --------------------------------------------------

channels:
  slack:
    enabled: false
    config: {}                    # Adapter-specific settings
  discord:
    enabled: false
    config: {}
  telegram:
    enabled: false
    config: {}
  twitter:
    enabled: false
    config: {}

# -- Communication Style ---------------------------------------

communication:
  style:
    default: balanced              # detailed | balanced | concise
    department_overrides:          # Optional per-department overrides
      development: concise
      marketing: detailed
    agent_overrides:               # Optional per-agent overrides
      strategist: detailed
      architect: concise

# -- Librarian Graph Integration --------------------------------

librarian:
  graph:
    enabled: false                   # Enable knowledge graph
    source_root: vault               # Root directory to index
    output_dir: vault/.graphify      # Output directory
    exclude: [05-inbox, .obsidian]   # Excluded directories
    model: claude-3-haiku-20240307   # LLM for entity extraction
    max_tokens_per_run: 80000        # Budget cap (input tokens)
    incremental: true                # Use SHA256 cache
    prompt_hint:
      enabled: false                 # Inject graph hint into prompts

# -- CLI Sections (not read by core runtime) -------------------

deployment:
  target: docker-compose          # docker-compose | terraform | manual

llm:
  defaultProvider: anthropic      # anthropic | openai | openrouter
  defaultModel: claude-sonnet-4-20250514

networking:
  apiPort: 3000                   # 1-65535

observability:
  logLevel: info                  # debug | info | warn | error
```

## Section Details

### storage

Configures backend adapters for each data store.

**storage.state** -- Primary document store for operators, audit logs, execution records, agent configs.

| Field | Type | Default | Env Fallback |
|-------|------|---------|-------------|
| `type` | `"mongodb"` | `"mongodb"` | -- |
| `uri` | string | -- | `MONGODB_URI` |
| `database` | string | `"yclaw_agents"` | `MONGODB_DB` |

**storage.events** -- Event transport for inter-agent communication, queues, KV store.

| Field | Type | Default | Env Fallback |
|-------|------|---------|-------------|
| `type` | `"redis"` | `"redis"` | -- |
| `url` | string | -- | `REDIS_URL` |

**storage.memory** -- Persistent agent memory (long-term knowledge, episodes, working memory).

| Field | Type | Default | Env Fallback |
|-------|------|---------|-------------|
| `type` | `"postgresql"` | `"postgresql"` | -- |
| `url` | string | -- | `MEMORY_DATABASE_URL` |

**storage.objects** -- File and asset storage (onboarding uploads, generated content).

| Field | Type | Default | Env Fallback |
|-------|------|---------|-------------|
| `type` | `"local"` or `"s3"` | `"local"` | -- |
| `path` | string (local only) | `./data/objects` | `YCLAW_OBJECT_STORE_PATH` |
| `bucket` | string (s3 only) | -- | `YCLAW_S3_BUCKET` |
| `prefix` | string (s3 only) | -- | -- |
| `region` | string (s3 only) | -- | `AWS_REGION` |

### secrets

Configures how credentials are loaded at bootstrap.

| Provider | Use Case |
|----------|----------|
| `env` | Reads from `process.env` (default, local/Docker Compose) |
| `aws-secrets-manager` | Reads from AWS Secrets Manager (production) |

For `aws-secrets-manager`:
- `prefix` -- Secret name prefix (default: `yclaw/`)
- `region` -- AWS region (default: `AWS_REGION` env or `us-east-1`)

### channels

Each channel adapter has `enabled` (boolean) and optional `config` (adapter-specific).

Credentials go in `.env`, not in `config`:

| Channel | Required Env Var |
|---------|-----------------|
| Slack | `SLACK_BOT_TOKEN` |
| Discord | `DISCORD_BOT_TOKEN` |
| Telegram | `TELEGRAM_BOT_TOKEN` |
| Twitter | `TWITTER_API_KEY`, `TWITTER_API_SECRET`, `TWITTER_ACCESS_TOKEN`, `TWITTER_ACCESS_SECRET` |

### communication

Controls how verbose AI agents are in their internal communications (status updates, handoffs, discussions). Does **not** affect deliverables like blog posts, docs, or customer-facing content.

**Allowed values:** `detailed`, `balanced`, `concise`

| Style | Description | Best For |
|-------|-------------|----------|
| `detailed` | Full sentences with context and reasoning | Executive, review, content roles |
| `balanced` | Clear and concise, no fluff | General default |
| `concise` | Brief fragments, maximum efficiency | Technical execution, monitoring |

**Resolution precedence (highest wins):**

1. Agent-level override (from agent YAML `communication.style` field)
2. Agent-level override (from `communication.style.agent_overrides` in main config)
3. Department-level override (from `communication.style.department_overrides`)
4. Global default (from `communication.style.default`)
5. Fallback: `balanced`

Each agent YAML can optionally include a `communication` section:

```yaml
name: architect
department: development
communication:
  style: concise    # Optional override
```

Code blocks, error messages, JSON, and technical terms are never affected by style.

### librarian

Configures the Librarian agent's knowledge graph integration via [graphify](https://github.com/safishamsi/graphify).

```yaml
librarian:
  graph:
    enabled: true
    source_root: vault
    output_dir: vault/.graphify
    exclude: [05-inbox, .obsidian]
    model: claude-3-haiku-20240307
    max_tokens_per_run: 80000
    incremental: true
    prompt_hint:
      enabled: false
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | `false` | Enable the knowledge graph |
| `source_root` | string | `vault` | Root directory to index |
| `output_dir` | string | `vault/.graphify` | Output directory for graph artifacts |
| `exclude` | string[] | `[05-inbox, .obsidian]` | Directories to exclude |
| `model` | string | `claude-3-haiku-20240307` | LLM model for entity extraction |
| `max_tokens_per_run` | number | `80000` | Max input tokens per run (budget cap) |
| `incremental` | boolean | `true` | Use SHA256 cache for incremental updates |
| `prompt_hint.enabled` | boolean | `false` | Inject graph hint into agent system prompts |

**What it does:**

- Runs graphify after Librarian files notes (daily) and after hygiene (weekly)
- Produces `graph.json` (queryable graph), `GRAPH_REPORT.md` (god nodes, communities), and `graph.html` (visualization)
- Exposes `vault:graph_query` action for agents to query graph relationships
- Enhances deduplication with graph community/neighbor signals
- Enriches standup reports with graph health metrics

**Cost control:**

- Always uses `--update` (incremental, SHA256 cache)
- Code files (`.py`, `.ts`, etc.) use AST-only extraction (no LLM)
- Budget cap: falls back to deterministic-only mode if exceeded (report marked `degraded`)
- Graph failure never crashes the curation pipeline

**Output files:**

| File | Committed | Purpose |
|------|-----------|---------|
| `vault/.graphify/graph.json` | Yes | Persistent queryable graph |
| `vault/.graphify/GRAPH_REPORT.md` | Yes | God nodes, communities, questions |
| `vault/.graphify/graph.html` | No (.gitignored) | Interactive visualization |
| `vault/.graphify/.cache/` | No (.gitignored) | SHA256 incremental cache |

**Prompt hint (disabled by default):**

When `prompt_hint.enabled: true`, agents receive a system prompt injection:

> "Before searching the vault, consult `vault/.graphify/GRAPH_REPORT.md` for structural overview — god nodes, communities, and cross-area connections. Use `vault:graph_query` for relationship questions."

### deployment (CLI only)

| Target | What Happens |
|--------|-------------|
| `docker-compose` | Generates and runs `docker-compose.yml` |
| `terraform` | Generates Terraform config for AWS |
| `manual` | Prints setup instructions |

| Field | Type | Default |
|-------|------|---------|
| `defaultProvider` | `anthropic`, `openai`, `openrouter` | `anthropic` |
| `defaultModel` | string | `claude-sonnet-4-20250514` |

### networking (CLI only)

| Field | Type | Default |
|-------|------|---------|
| `apiPort` | integer (1-65535) | `3000` |

### observability (CLI only)

| Field | Type | Default |
|-------|------|---------|
| `logLevel` | `debug`, `info`, `warn`, `error` | `info` |

## Defaults

If no `yclaw.config.yaml` exists, the core runtime falls back to environment variables:

```
MONGODB_URI -> mongodb://localhost:27017
MONGODB_DB -> yclaw_agents
REDIS_URL -> redis://localhost:6379
MEMORY_DATABASE_URL -> (optional)
```

This preserves backward compatibility with pre-CLI setups.

---

## Validation

```bash
yclaw config validate              # Basic validation
yclaw config validate --strict     # Warn on missing deployment/channels
```

The schema is enforced with Zod `.strict()` mode -- unknown keys are rejected. The core runtime reads `storage`, `secrets`, `channels`, `communication`, and `librarian`. The CLI reads the additional `deployment`, `llm`, `networking`, and `observability` sections.
